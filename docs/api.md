# API-documentatie

De REST API staat volgens de PRD in fase 2. Voor het MVP loopt alle interactie via server-side webformulieren en sessies.

Geplande endpoints voor fase 2:

- `GET /api/tickets`
- `POST /api/tickets`
- `GET /api/tickets/{id}`
- `PATCH /api/tickets/{id}`
- `POST /api/tickets/{id}/replies`

Authenticatie, rate limiting en API-tokens worden pas uitgewerkt wanneer fase 2 wordt gestart.
